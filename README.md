
# Предсказание сепсиса на основе архитектур LSTM, Mamba и Transformer

Проект реализует полный исследовательский пайплайн для раннего предсказания сепсиса на основе данных интенсивной терапии. Ключевые направления работы: сравнение нейросетевых архитектур (LSTM, Mamba, Transformer), разработка методов борьбы с дисбалансом классов, проверка устойчивости моделей к синтетическим сдвигам данных, валидация на датасете PhysioNet Challenge 2019.

## 1. Постановка задачи

## 1.1. Основная цель исследования
Исследовать применимость архитектуры Mamba (State Space Model) к задаче классификации медицинских временных рядов на примере раннего предсказания сепсиса у пациентов отделений интенсивной терапии.
# 1.2. Формулировка задачи
Дана последовательность многомерных наблюдений пациента во времени:
X={x_1,x_2,...,x_T},x_t ∈ R^D ∪ {NaN}
где:
T — длина последовательности (часы наблюдения)
D=40 — количество признаков (витальные показатели, лабораторные анализы)
NaN — пропущенные значения (до 80% данных)
Требуется построить модель f:X→[0,1], которая предсказывает вероятность наличия сепсиса у пациента в текущий момент времени:
y^ = f(X) ∈ [0,1], y ∈ {0,1}

## 3. Предобработка данных
# 3.1. Обработка пропущенных значений
Пропущенные значения в медицинских данных несут информацию о процессе наблюдения (например, анализ не назначен, потому что врач не увидел необходимости). Простое удаление или заполнение средним может исказить распределение.
Принятый подход: метод маскирования пропусков, предложенный в работе Che et al. (2018) "Recurrent Neural Networks for Multivariate Time Series with Missing Values"
Обоснование:
Маска позволяет модели отличать "реальные нули" от "заполненных пропусков"
Нормализация только на train предотвращает утечку информации из val/test
Заполнение нулями не искажает распределение, так как маска обнуляет вклад этих значений
# 3.2. Нормализация признаков
Применена стандартизация (Z-score normalization) на основе статистик тренировочной выборки:
x_norm=x−μ_train/σtrain + ϵ, где ϵ= 10^−8 для численной стабильности.
Важно: статистики 
μ_train, σ_train вычисляются только на тренировочных данных и применяются ко всем сплитам для предотвращения утечки.
# 3.3. Формирование последовательностей
Для каждого пациента формируем фиксированную последовательность длины 
T=48:
Если записей > 48: берём последние 48 часов (наиболее релевантные для прогноза)
Если записей < 48: дополняем нулями с маскированием
## 4. Архитектуры моделей
# 4.1. LSTM (Long Short-Term Memory) — базовая архитектура
Параметры: 569,000
Преимущества: Проверенная архитектура для временных рядов, устойчива к пропускам
Недостатки: Квадратичная сложность по памяти при длинных последовательностях
# 4.2. Mamba (State Space Model) — исследуемая архитектура
Параметры: 69,000 (на 88% меньше LSTM)
Преимущества:
Линейная сложность O(n) против O(n^2)у Transformer
Эффективный захват длинных зависимостей через state space representation
Меньшее количество параметров при сопоставимом качестве
Недостатки: Новая архитектура, меньше опубликованных результатов на медицинских данных
# 4.3. Transformer — архитектура для сравнения
Параметры: 565,000
Преимущества: Параллелизация, мощный механизм внимания для глобальных зависимостей
Недостатки: Чувствительность к пропускам, требует больше данных для стабильного обучения

6.1 Основные метрики
| Метрика | Формула | Интерпретация | Приоритет для задач |
|---------|---------|---------------|---------------------|
| AUROC | Площадь под ROC-кривой | Способность ранжировать положительные примеры выше отрицательных | Высокий — сравнение моделей независимо от порога |
| F1-мера | \( \frac{Precision \cdot Recall}{Precision + Recall} \) | Гармоническое среднее точности и полноты | Высокий — оптимизация порога классификации |
| Precision | \( \frac{TP}{TP + FP} \) | Доля верных предсказаний сепсиса среди всех предсказанных | Критический — минимизация ложных тревог |
| Recall | \( \frac{TP}{TP + FN} \) | Доля обнаруженных случаев сепсиса среди всех реальных | Высокий — не пропустить опасные состояния |
| Accuracy | \( \frac{TP + TN}{TP + TN + FP + FN} \) | Общая доля правильных предсказаний | Низкий — вводит в заблуждение при дисбалансе |
6.2. Оптимизация порога классификации
Проблема стандартного порога 0.5
При сильном дисбалансе классов стандартный порог классификации 
0.5 приводит к неоптимальному балансу между точностью и полнотой:
Пример для LSTM при пороге 0.5:
• Precision = 0.889  (11% ложных тревог)
• Recall = 0.842     (16% пропущенных случаев)
• F1 = 0.865
Метод подбора оптимального порога
Оптимальный порог определялся на валидационной выборке (2,033 пациента) путём максимизации F1-меры:
from sklearn.metrics import precision_recall_curve

# Получаем кривые точности-полноты
precision, recall, thresholds = precision_recall_curve(y_val, y_prob)

# Вычисляем F1 для каждого порога
f1_scores = 2 * precision * recall / (precision + recall + 1e-8)

# Находим порог с максимальным F1
optimal_idx = np.argmax(f1_scores)
optimal_threshold = thresholds[optimal_idx] if len(thresholds) > 0 else 0.5
# Сравнение моделей классификации

| Модель | Оптимальный порог | F1 | Precision | Recall |
|--------|-------------------|----|-----------|--------|
| LSTM v2 | 0.9404 | 0.900 | 0.984 | 0.829 |
| Mamba v3 | 0.9430 | 0.837 | ~0.92 | ~0.76 |
| Transformer v2 | 0.9395 | 0.770 | 0.881 | 0.684 |

Наблюдение: Transformer при пороге 0.5 предсказывал сепсис слишком часто — из всех предсказаний "сепсис" только 31% оказывались верными. Это указывало на плохую калибровку вероятностей: выходные значения модели не соответствовали реальной уверенности.
Гипотеза и метод решения
Гипотеза: Низкая точность Transformer при пороге 0.5 объясняется не архитектурными ограничениями, а смещённым распределением предсказанных вероятностей. Оптимизация порога может выровнять сравнение моделей.
Метод: Порог классификации подбирался на валидационной выборке (2,033 пациента) путём максимизации F1-меры:
from sklearn.metrics import precision_recall_curve

# Кривые точности-полноты для валидационных предсказаний
precision, recall, thresholds = precision_recall_curve(y_val, y_prob)

# F1 для каждого порога
f1_scores = 2 * precision * recall / (precision + recall + 1e-8)

# Оптимальный порог
optimal_idx = np.argmax(f1_scores)
optimal_threshold = thresholds[optimal_idx] if len(thresholds) > 0 else 0.5

После успешной оптимизации порога для Transformer, та же процедура была применена ко всем моделям для обеспечения сравнения в равных условиях:

# 9.1. Методология повышения чувствительности модели
Стандартные алгоритмы федеративного обучения (FedAvg) склонны игнорировать миноритарный класс (сепсис) из-за сильного дисбаланса данных (соотношение здоровых к больным ≈ 25:1). Для решения этой проблемы и достижения клинически значимого качества были применены две ключевые стратегии:
Взвешенная функция потерь (Weighted BCE Loss):
На уровне каждого локального клиента была применена модифицированная функция потерь с коэффициентом веса положительного класса 
wpos=15.
это заставляет модель штрафовать ошибки на пациентах с сепсисом в 15 раз сильнее, чем на здоровых, компенсируя дисбаланс распределения.
Адаптивная калибровка порога классификации:
Вместо стандартного порога вероятности 0.5, оптимальное значение порога T∗ было выбрано на валидационной выборке путем максимизации метрики F1-score. Это позволяет найти баланс между точностью (Precision) и полнотой (Recall), наиболее подходящий для задач скрининга.

Федеративная модель LSTM, обученная с использованием взвешенной функции потерь и адаптивного порога, достигает качества, достаточного для внедрения в качестве системы поддержки принятия решений (CDSS). Потеря качества по сравнению с централизованным обучением (≈15% по AUROC) является обоснованной платой за соблюдение строгих требований к конфиденциальности медицинских данных.
